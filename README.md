# 🎙️ Real-Time Conversation Analytics

### Hackathon Project – Novara 2026 [Live Demo](https://aiconversationalanalytics.netlify.app/)

## 📌 Overview

This project implements a **Real-Time Conversation Analytics System** that processes **streaming transcripts generated by an Automatic Speech Recognition (ASR) system** and extracts meaningful insights from the conversation as it happening.

In many real-world applications such as **customer support platforms, call centers, and AI voice assistants**, it is extremely valuable to analyze conversations **while they are happening**. Real-time insights allow organizations to:

* Understand **customer intent**
* Detect **sentiment and dissatisfaction**
* Identify **escalation risks**
* Monitor **service quality**

This project was developed during the **Novara 2026 Hackathon** to demonstrate how **Large Language Models (LLMs)** can be integrated with **real-time speech systems** to provide structured conversational intelligence.

---

# 🚀 Key Features

### 🔹 Real-Time Transcript Processing

Continuously receives **streaming ASR-generated text segments** and processes them dynamically.

### 🔹 Intent Detection

Identifies the **purpose of the speaker’s message**, such as:

* Complaint
* Query
* Greeting
* Request

### 🔹 Sentiment Analysis

Determines the emotional tone of the conversation:

* Positive
* Neutral
* Negative

### 🔹 Escalation Detection

Identifies potential **customer frustration or escalation risks** that may require human intervention.

### 🔹 Topic Classification

Detects the **subject of the conversation**, helping organizations categorize interactions.

### 🔹 Dynamic Analytics

The system **updates insights continuously** as the conversation progresses.

# 🧠 System Architecture

The system integrates **ASR streaming, backend APIs, orchestration tools, and LLMs**.

```
ASR System
   │
   │ (Streaming Transcript)
   ▼
FastAPI Backend (Python)
   │
   │ Sends text segments
   ▼
n8n Workflow Engine
   │
   │ Orchestrates LLM calls
   ▼
Grok API (LLM)
   │
   │ Extracts structured insights
   ▼
Conversation Analytics Output
(Intent, Sentiment, Topic, Escalation Risk)
```

# ⚙️ How It Works

### 1️⃣ ASR Streaming

The system receives **live transcript segments** from an ASR engine.

Example input:

```
Hello I need help with my order
```

---

### 2️⃣ Backend Processing

A **FastAPI backend service** receives the transcript segments and forwards them to the analytics pipeline.

---

### 3️⃣ LLM Orchestration (n8n)

The **n8n workflow** manages communication with the LLM by:

* Receiving transcript segments
* Triggering the Grok API
* Processing the model response

---

### 4️⃣ LLM Analysis

The LLM extracts structured insights such as:

```json
{
  "intent": "Customer Support Request",
  "sentiment": "Neutral",
  "escalation_risk": "Low",
  "topic": "Order Issue"
}
```

---

### 5️⃣ Real-Time Updates

As more transcript segments arrive, the system **updates analytics dynamically**.

# ▶️ Running the Project

### 1️⃣ Clone the repository

```bash
git clone https://github.com/yourusername/conversation-analytics.git
cd conversation-analytics
```

---

### 2️⃣ Create Virtual Environment

```bash
python -m venv venv
source venv/bin/activate
```

Windows:

```bash
venv\Scripts\activate
```

---

### 3️⃣ Install Dependencies

```bash
pip install -r requirements.txt
```

---

### 4️⃣ Start FastAPI Server

```bash

```

---

Run the n8n workflow and configure the **Grok API integration**.
